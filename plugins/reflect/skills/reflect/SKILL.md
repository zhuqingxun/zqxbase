name: reflect
description: 复盘当前会话，提取可沉淀的经验。当用户要求复盘、总结经验、提取教训、回顾会话收获时使用
---

复盘本次会话，分析以下内容：

1. **错误扫描（必须首先执行）**：回溯会话中所有失败的工具调用（Bash exit code 非 0、Edit 被拒绝、API 报错等），逐一分析：
   - 该错误是否暴露了现有规则的缺陷或盲区？
   - 该错误是否违反了已有的 CLAUDE.md 规则？（如果是，说明规则的表述不够强或触发条件不够宽）
   - 快速修复的错误也不能跳过——"一次迭代修好"不等于"不值得反思"
2. **识别关键经验**：回顾会话中的任务，找出成功解决的问题和遇到的障碍
3. **提取通用原则**：将具体经验抽象为可复用的规则（技术要点、工作流程、注意事项）
4. **简洁表达**：规则要足够简洁（3-5行），同时具有指导性

对每条经验，判断其适用范围并建议写入位置：

| 级别 | 写入位置 | 判断标准 | 效果 |
|------|----------|----------|------|
| **全局规则** | `~/.claude/CLAUDE.md` | 跨所有项目通用的强制约束（如 bash 语法、git 规范、编码风格） | 每次会话强制加载，必须遵守 |
| **项目规则** | 项目根目录 `CLAUDE.md` | 仅限当前项目的开发规范（如架构约定、特定 API 用法） | 在该项目目录下工作时强制加载 |
| **Auto Memory** | `~/.claude/projects/*/memory/MEMORY.md` | 参考性经验、调试技巧、历史决策记录，不需要强制执行 | 每次会话加载为上下文参考 |

输出格式：
- 列出发现的经验点，每条标注建议级别
- 给出拟写入的简洁规则文本
- 使用 AskUserQuestion 让用户确认每条经验的写入位置（可选：写入建议位置 / 换位置 / 跳过）
- 用户确认后写入对应文件

注意：只提取具有通用性、可复用的经验，忽略一次性或特定场景的内容。
